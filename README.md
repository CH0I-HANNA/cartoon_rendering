# Project Name: Cartoon Rendering

## 1. 프로그램 기능 설명
본 프로그램은 사용자로부터 입력받은 이미지를 분석하여 다음과 같은 기능을 수행합니다.

* **노이즈 억제 및 전처리**: 에지 검출 시 발생하는 자잘한 잡음을 제거하기 위해 `Median Blur`를 적용하여 전반적인 선의 깔끔함을 확보합니다.
* **적응형 외곽선 검출**: 조명 조건이나 이미지의 국부적인 밝기 변화에 유연하게 대응하는 `Adaptive Thresholding` 기법을 사용하여 사물의 뚜렷한 윤곽선을 추출합니다.
* **에지 보존 색상 단순화**: `Bilateral Filter`를 사용하여 사물의 경계는 유지하면서도 내부의 세밀한 질감(땀, 근육, 주름 등)을 뭉뚱그려 평면적인 만화적 색감을 구현합니다.
* **실시간 결과 비교 (Side-by-Side)**: `cv2.hconcat` 기능을 활용하여 원본 이미지(Before)와 변환된 이미지(After)를 하나의 창에 나란히 출력함으로써 알고리즘의 효과를 직관적으로 확인하게 합니다.
* **자동 파일 저장**: 변환된 최종 카툰 이미지와 비교용 스크린샷 이미지를 로컬 디렉토리에 자동으로 저장하여 보고서 활용성을 높였습니다.

* **대상**: 캡틴 아메리카 피규어 및 역동적인 복싱 경기 사진
* **분석**:
    * **선명한 윤곽선**: 피사체와 배경의 대비가 명확한 피규어 이미지에서 `adaptiveThreshold`가 객체의 외곽선을 굵고 선명하게 잡아내어 완벽한 애니메이션 느낌을 구현했다.
    * **색상 단순화**: 복싱 선수의 경기복이나 피부의 미세한 명암 단계가 `bilateralFilter`에 의해 과감하게 뭉뚱그려져, 실제 사진의 질감 대신 만화 특유의 '색면(Flat Color)' 느낌이 극대화되었다.
    * **노이즈 억제**: 경기장 배경의 복잡한 요소들이 `medianBlur`를 통해 정돈되어 지저분한 검은 점 없이 깔끔하게 렌더링되었다.

## 3. 알고리즘의 한계점 논의

1.  **텍스처와 에지의 구분 한계**: 알고리즘이 의미 있는 '구조적 경계'와 단순한 '피부 질감'을 완벽히 구분하지 못해 디테일이 많은 사진에서 결과물이 지저분해진다.
2.  **고정 파라미터 의존성**: 이미지의 해상도에 상관없이 고정된 `blockSize`를 사용하므로, 사진의 크기에 따라 선의 굵기가 일정하지 않은 문제가 있다.
3.  **연산 복잡도**: `bilateralFilter`는 연산량이 많아 고해상도 영상의 실시간 렌더링 시 프레임 드랍이 발생할 가능성이 크다.
4.  **조명 및 대비 민감도**: 대비가 낮은 이미지에서는 카툰 특유의 명확한 구분감이 사라지고 결과물이 밋밋해지는 공학적 한계가 존재한다.
